Product Plan

새 책 읽기 플로우

PDF 없이 책을 읽기 시작할 때, 사용자가 먼저 책 컨텍스트를 고르고 그 안에서 인용문만 빠르게 입력하도록 만드는 기획안.

결정된 방향

첫 화면은 가장 최근 인용문을 추가한 책을 보여준다. 저자 목록의 최상단에는 새 책 읽기 진입점을 둔다. 사용자는 책 제목과 저자를 입력한 뒤 그 책의 읽기 화면으로 들어간다.

책 컨텍스트가 화면 단위로 고정되므로, 인용문 입력 칸에서는 author와 book 필드를 제거한다. 입력자는 인용문과 페이지에 집중한다.

문제

  • 현재 PDF 없이 새 책을 시작하는 명시적 버튼이 없다.
  • 책은 첫 인용 저장 시 암묵적으로 생긴다.
  • 인용 입력창마다 저자와 책 제목을 반복 입력해야 한다.
  • 빈 책 상태가 없어 “읽기 시작”이라는 사용자의 의도가 UI에 남지 않는다.

완료 기준

  • 앱 진입 시 최근 책 화면이 열린다.
  • 저자 목록 최상단에서 새 책 읽기를 시작할 수 있다.
  • 책 제목과 저자를 입력하면 빈 Book View로 진입한다.
  • Book View 입력창에는 인용문과 페이지 입력만 남는다.
  • 첫 인용 저장 후 Library에 책이 안정적으로 표시된다.

사용자 흐름

1

앱 첫 화면

최근에 인용문을 추가한 책의 Book View를 기본으로 연다. 최근 책이 없으면 전체 Archive를 연다.

2

새 책 읽기 진입

오른쪽 Library 영역의 저자 목록 최상단에 새 책 읽기 버튼을 둔다.

3

책 정보 입력

모달에서 저자, 책 제목, 시작 페이지를 입력한다. 시작 페이지는 선택값으로 둔다.

4

Book View 시작

빈 Book View로 이동한다. 상단 제목은 책 제목, 보조 정보는 저자를 보여준다.

5

인용문 추가

입력창은 인용문과 페이지 중심으로 축소한다. 저장 시 현재 Book View의 저자와 책 제목을 자동 사용한다.

상태 모델

기본 상태 selectedBookContext가 있으면 Book View, 없으면 Archive View.
최근 책 인용문의 createdAt 기준으로 가장 최근 인용이 속한 책을 계산한다.
새 책 초안 저자와 책 제목을 가진 클라이언트 상태. 첫 인용 전까지는 DB 책 레코드 생성 여부를 별도 결정한다.
저장 인용 저장 시 현재 책 컨텍스트를 author, book 값으로 주입한다.
복귀 사용자가 다른 저자/책을 선택하면 기존 Library 필터 동작과 동일하게 Book View를 전환한다.

화면 변경

  • Library 상단: 새 책 읽기 버튼 추가.
  • Archive 기본 화면: 최근 책 자동 선택.
  • Book View 헤더: 책 제목, 저자, 정렬 버튼 유지.
  • Book View 입력창: 저자/책 필드 제거, 페이지 필드는 유지.
  • 전체 Archive 입력창: 저자/책 필드는 유지하거나 숨김 정책을 별도 결정.

데이터 변경

  • 필요: 최근 책 계산 로직.
  • 필요: 새 책 초안 상태.
  • 필요: 현재 책 컨텍스트를 인용 저장 입력에 병합하는 경로.
  • 미정: 빈 책 레코드를 즉시 생성할지, 첫 인용 시 생성할지.

결정 필요

아래 항목은 구현 전 결정이 필요하다. 상태 구조와 데이터 책임에 영향을 준다.

  • 빈 책 생성 시점: 첫 인용 전 DB에 책을 만들지, 아니면 클라이언트 초안으로만 둘지.
  • 전체 Archive 입력창: 전체 화면에서는 저자/책 필드를 유지할지, 새 책 흐름 이후 전역적으로 제거할지.
  • 모바일: 새 책 읽기 버튼을 모바일 첫 화면에도 넣을지, 데스크톱 우선으로 둘지.

위험

Data UX
  • 현재 Library 트리는 인용문에서 책을 파생한다. 빈 책을 즉시 만들면 트리 쿼리 구조가 바뀐다.
  • 첫 인용 전 초안 상태는 새로고침 시 사라질 수 있다.
  • 저자/책 필드를 제거하면 잘못 들어간 책 컨텍스트를 빠르게 수정하는 경로가 필요하다.

검증 기준

  • 첫 진입 시 최근 책이 선택된다.
  • 새 책 읽기 후 빈 Book View가 열린다.
  • 인용 저장 시 저자/책이 자동으로 붙는다.
  • 저장 후 Library에 해당 책이 보인다.
  • 기존 PDF Reader 저장 흐름은 깨지지 않는다.

1차 구현 범위

  1. 최근 책 계산과 초기 선택 로직 추가.
  2. Library 최상단에 새 책 읽기 버튼 추가.
  3. 새 책 입력 모달 추가.
  4. Book View용 간소 입력창 변형 추가.
  5. 현재 책 컨텍스트 기반 인용 저장 연결.
  6. 단위 테스트와 브라우저 흐름 검증.
